# 公众号论文长图 - Prompt

---

## HTML 长图生成（给 Claude Code）

````
你是一名微信公众号长图排版设计师和前端开发助手。请根据我接下来提供的用户已确认文案稿、论文信息、PDF论文和图表素材，生成一个适合截图导出为公众号长图的完整 HTML 文件。

## 零、输入前提（不要跳过）

生成 HTML 前必须确认：

1. 用户已经修改并确认文案稿。
2. PDF 图表已经提取到 `assets/`，并确认每张 `assets/N.png` 和原论文 Fig 的对应关系。
3. 已确认配色方案。
4. `assets/top.png` 已存在。

HTML 内容必须以用户确认后的文案稿为准。不要擅自改动核心表达、实验顺序、结论强度或图表对应关系。只在不改变事实和结构的前提下做适合长图排版的轻微压缩和分段。

## 一、总体目标

请生成一个完整的单文件 HTML 页面，可以直接保存为 .html 文件并在浏览器中打开。页面最终用途是截图生成微信公众号长图，优先保证手机端视觉效果、纵向阅读节奏、模块化结构和截图稳定性。

页面风格不是传统论文排版，而是年轻化、信息图式公众号推文风格：高饱和配色、模块化卡片、粗体标题、圆角边框、贴纸感装饰、数据卡片、简洁图表说明、强视觉封面。

请特别注意：如果文章包含多个实验或多个研究，必须让"实验内容"和"该实验对应的图表"放在同一个 section 框里。不要把所有图表集中放到文章后面的"图表展示区"。

## 二、封面区处理

封面区使用本地路径 `assets/top.png` 作为背景图，直接写 `<img>` 标签，不生成占位区域。

请按以下结构使用封面图：

```html
<section class="section cover">
  <img src="assets/top.png" alt="封面背景图" style="width:100%;display:block;position:absolute;inset:0;object-fit:cover;">
  <div class="cover-overlay">
    <!-- 在此之上叠加标题、副标题、标签等文字内容 -->
  </div>
</section>
```

封面图是一张概念型视觉背景图，已在画面中为标题留出了空间。你需要：
- 将标题、副标题、标签等内容通过 `.cover-overlay` 叠加在封面图之上
- 如果图片本身偏亮，标题用深色；如果偏暗，标题用浅色
- 可以通过半透明遮罩层增强文字可读性
- 封面区整体作为第一屏，要有冲击力
- **封面区高度控制在 280–340px 以内**，使用紧凑布局。即使封面图缺失，文字区域也不会因大片空白而显得突兀

## 三、页面尺寸与基础要求

1. 页面为手机端长图预览，主容器宽度固定为 390px，居中显示。
2. body 背景可以是深色或浅色，但主内容容器需要有统一底色。
3. 所有 CSS 必须写在 HTML 内部的 <style> 标签中。
4. 不使用 JavaScript。
5. 不引用外部 CSS、字体库、图标库或在线资源。
6. 页面必须适合用浏览器截图或 Playwright 截图导出 PNG。
7. 每个内容模块都必须使用 <section class="section 模块名"> 结构，方便后续按 section 分别导出图片。
8. 每个 section 必须有明显留白，避免内容贴边。
9. 每个 section 建议使用圆角、边框、阴影或色块区分。
10. section 之间的视觉分割线必须写在 section 内部底部（使用 border-bottom），不要使用独立的 `<div class="section-divider">`。封面区和最后一个 section 不需要底线。深色背景的 section 使用对应的浅色底线。否则 Playwright 按 section 截图时分割线会出现在顶部看起来很突兀。
11. 所有图片使用本地相对路径，按出现顺序从1开始编号：
    <img src="assets/top.png" alt="封面背景图">
    <img src="assets/1.png" alt="图表1">
    <img src="assets/2.png" alt="图表2">
    ...
    <img src="assets/N.png" alt="图表N">
12. 所有图表位置直接写 `<img>` 标签，不要生成任何占位区域（如 figure-placeholder），不要写"此处放置图表"等占位文字。
13. 根据已确认的图表对应关系，在 HTML 中使用正确编号的 img 标签。

## 四、最重要的结构规则：实验与图表必须绑定

如果原文包含多个实验、多个研究、多个 Study、Experiment 或 Figure，请严格遵守：

1. 同一个实验的全部内容必须放在同一个 section 里。
2. 与该实验相关的图片、图表、流程图、结果图，必须放在该实验 section 内部。
3. 图片必须紧跟在对应实验的文字说明之后，而不是集中放在文章最后。
4. 不要单独设置一个总的"图表展示区"来堆放所有图片。
5. 如果一个实验对应多张图，则这些图都放在该实验 section 内部，并按解释顺序排列。
6. 如果某张图同时涉及多个实验，请放在最主要对应的实验 section 中，并在图注中说明。
7. 每个实验 section 的推荐内部顺序为：

   - section-kicker：实验编号，例如"04 实验1"
   - section-title：该实验的核心问题
   - 简短引入：这个实验为什么要做
   - flow-card 或 result-card：对象、操纵、任务、指标、发现
   - figure-card：该实验对应图表，使用 `<img src="assets/N.png">`（N为图片序号）
   - fig-title：中文图题
   - fig-caption：原论文 Fig. X / Extended Data Fig. X 信息
   - fig-explain：用公众号读者能理解的话解释图表重点

8. 不要出现这种结构：

   实验1文字
   实验2文字
   实验3文字
   图1
   图2
   图3

   正确结构应该是：

   实验1文字 + 图1
   实验2文字 + 图2
   实验3文字 + 图3

## 五、模块结构要求

请至少包含以下模块，并且每个模块都用 section 包裹。

### 1. 强视觉封面区

class 建议为：section cover

要求：
- 包含醒目的主标题、副标题、关键词标签、视觉装饰。
- 标题要有冲击力，适合公众号开头。
- 可以使用 emoji、几何图形、贴纸式标签，但不要过度堆砌。
- 封面区使用 assets/top.png 作为背景图，叠加文字内容。
- 封面文字区域必须可读，必要时通过遮罩、渐变、半透明蒙层等方式增强标题对比度。

推荐封面区内部结构示例：

<section class="section cover">
  <img src="assets/top.png" alt="封面背景图" style="width:100%;display:block;position:absolute;inset:0;object-fit:cover;">
  <div class="cover-overlay">
    <div class="cover-badge">心理学科普 / 论文解读</div>
    <h1>主标题</h1>
    <p class="subtitle">副标题</p>
    <div class="tag-row">关键词标签</div>
  </div>
</section>

### 2. 问题引入区

class 建议为：section intro

要求：
- 用生活化场景引入研究问题。
- 语言适合公众号读者，但不要过度口语化。
- 可以用"你是不是也……"或"我们常常……"开头。
- 结尾要自然过渡到论文研究问题。

### 3. 研究设计总览区

class 建议为：section design overview

要求：
- 用信息流方式解释整篇研究的总体逻辑。
- 可以使用流程卡片，例如：
  话题/材料选择 → 条件操纵 → 对话/任务 → 结果指标。
- 不要写成大段方法学论文。
- 如果后面会逐个介绍实验，这里只做总览，不要重复所有细节。

### 4. 逐个实验区

class 建议为：section experiment

要求：
- 每个重要实验或 Study 单独使用一个 section。
- 同一个实验的介绍、方法、发现、对应图片必须放在同一个 section 里。
- 如果实验很多，可以筛选 3–5 个最适合公众号读者理解的重点实验。
- 每个实验建议包括：
  - 这个实验想回答什么问题
  - 被试或样本
  - 实验怎么做
  - 比较了什么条件
  - 测量了什么结果
  - 发现了什么
  - 对应图表和图表解释

推荐结构示例：

<section class="section experiment">
  <span class="section-kicker">04 实验1</span>
  <h2 class="section-title">实验1标题：用一句话说清核心问题</h2>

  <p>简短说明这个实验为什么要做。</p>

  <div class="flow">
    <div class="flow-card">
      <span class="step">对象</span>
      <h3>被试/样本</h3>
      <p>简要说明参与者来源和数量。</p>
    </div>

    <div class="flow-card">
      <span class="step">做法</span>
      <h3>实验任务或操纵</h3>
      <p>说明实验如何进行。</p>
    </div>

    <div class="flow-card">
      <span class="step">发现</span>
      <h3>核心结果</h3>
      <p>用公众号读者能理解的话解释结果。</p>
    </div>
  </div>

  <div class="figure-card">
    <img src="assets/1.png" alt="原论文 Figure 1：实验1结果图" style="width:100%;display:block;border-radius:12px;">
    <div class="fig-title">图1：中文图题</div>
    <p class="fig-caption">原论文 Fig. 1。说明这张图对应实验1，展示了什么结果。</p>
    <div class="fig-explain">读图重点：用一两句话解释读者应该看哪里、说明了什么。</div>
  </div>
</section>

### 5. 核心结果整合区

class 建议为：section result

要求：
- 在逐个实验介绍之后，用 2–4 个结果卡片总结整体发现。
- 每个结果卡片包含：标号 / 小标题 / 简短解释。
- 如果有关键统计结果，可以用醒目的数据标签呈现。
- 这里是综合总结，不要再重复堆放实验图表。

### 6. 研究启示区

class 建议为：section insight

要求：
- 把研究结论转化为现实生活、学习、工作、心理健康或社会议题中的启示。
- 使用卡片或短段落，不要写得像论文讨论部分。
- 可以加入一句核心金句。
- 不要过度鸡汤化，不要把相关结果夸大成因果结论。

### 7. 总结区

class 建议为：section summary

要求：
- 用 3–5 句话总结全文。
- 结尾要有记忆点，但不要标题党、不要鸡汤化。
- 如果我明确要求公众号关注区，再添加；如果没有要求，不要主动添加二维码或关注区。

### 8. 论文/研究信息区

class 建议为：section paper reference

要求：
- 简要介绍论文题目、作者、年份、期刊和核心问题。
- 如果是英文论文，请给出中文解释。
- 文献信息要简洁，但要准确。
- 可以使用信息卡片展示作者、期刊、年份、样本、研究数量等。
- 论文信息区必须放在文章最后，作为来源/参考信息，不要放在问题引入或研究设计之前。

## 六、图表处理要求

**重要提醒：生成 HTML 时，必须使用用户已确认的文案稿和图表对应关系。**

如果我提供 PDF 论文或图表素材，请按以下规则处理：

1. 先判断每张图对应哪个实验或研究。
2. 图表编号不能错。例如 Fig. 1 不能误放到实验2，Fig. 4 不能误放到实验3。
3. 图表必须紧跟对应实验，而不是放到文章后面。
4. 图表说明必须包含：
   - 中文图题
   - 原论文图号，例如"原论文 Fig. 1"
   - 对应实验，例如"对应实验1"
   - 读图重点
5. 所有图表使用从1开始的连续编号：`assets/1.png`、`assets/2.png`、`assets/3.png`… 按在文章中出现的顺序编号。
6. 直接写 `<img src="assets/N.png">` 标签，不要生成占位区域，不要写"此处放置图表"之类的占位文字。图片文件后续我会按编号放入 assets 目录。
7. 不要自己伪造图表数据。
8. 不要用 CSS 画一个看起来像真实论文结果的假图。
9. 图片容器要清楚、干净，适合手机阅读。
10. 图片不要铺得太满，建议放在圆角卡片内（img 外层用 .figure-card 包裹）。

## 七、视觉风格要求

请使用以下风格原则：

1. 标题字体加粗，层级明显。
2. 使用高饱和色，但整体要协调。
3. 每个模块之间有明显间距。
4. 卡片可以使用：
   - border-radius: 24px 或更大
   - border: 2px/3px solid #111827
   - box-shadow: 6px 6px 0 #111827
5. 避免纯论文式黑白排版。
6. 可以使用 emoji 作为视觉符号，但不要过多。
7. 图表区域要清楚、干净、适合手机阅读。
8. 正文行距建议 1.7–1.9。
9. 正文段落（`<p>` 标签）首行缩进 2em（`text-indent: 2em`），标题（h1/h2/h3）和卡片内的短文本不需要缩进。
10. 字号建议：
   - 封面标题：36–44px
   - 一级模块标题：24–28px
   - 实验小标题：17–20px
   - 正文：14–16px
   - 图注：11–13px
11. 避免模块内容过高；如果某一部分太长，请拆成多个 section。
12. 但是注意：如果是同一个实验，即使内容较长，也优先保持在同一个 section 中；可以通过内部卡片分层，而不是把图表拆出去。

### 配色方案速查

**浅色暖调**：米白底 + 深棕文字 + 琥珀强调
**深色科技**：深黑底 + 浅灰文字 + 霓虹强调
**高饱和活泼**：彩色卡片 + 粗边框 + 强阴影
**极简杂志**：大量留白 + 衬线标题 + 细线
**心理学科普**：柔和色块 + 圆角卡片 + 温暖调
**赛博霓虹**：暗底 + 荧光绿/紫 + 终端感
**暖调文艺杂志**：米白/浅奶油底色交替 + 1px 细线分隔 + 矩形卡片无圆角 + 深棕标题 + 灰褐正文 + 黄铜色（#c4b99a）为唯一强调色 + 段落首行缩进 + 大量留白 + 柔和字重层级。整体像一本有质感的纸质刊物，不争不抢，舞台让给文字本身。
**日报快讯**：纸媒专栏感 + 多栏或宽栏排版 + 粗衬线大标题 + 细分割线 + 紧凑信息密度。像一份有分量的科学简报，适合颠覆性发现。
**日系素净**：冷灰底 + 藏蓝文字 + 极细边框（0.5–1px）+ 字号偏小 + 极致留白。比极简杂志更冷、更克制，适合冥想、正念、认知相关主题。
**学术手账**：手写感标题字体 + 虚线边框 + 便签色块（淡黄、淡粉、淡绿）+ 手绘箭头和圈注装饰。像研究生在笔记本上做的精读笔记，适合方法论或综述类文章。
**暗夜阅读**：深灰底（#1a1a1e）而非纯黑 + 暖白文字（#e8e4dd）+ 低饱和暖色点缀（暗金、砖红）+ 柔和对比度。比深色科技温和，比赛博霓虹安静，适合严肃话题的夜间阅读。
**自然有机**：鼠尾草绿 + 陶土色 + 亚麻白 + 卡片边角微圆（8–12px）+ 色块模拟自然纸张肌理。适合生态、环境、进化心理学主题。

9. 图片路径使用连续编号：封面为 assets/top.png，图表按出现顺序为 assets/1.png、assets/2.png、assets/3.png...
10. 不要使用外部依赖。

## 九、后续导出 PNG 的兼容要求

为了方便我用 Playwright 把每个模块单独导出为 PNG，请严格遵守：

1. 每个主要模块必须是一个独立的 <section>。
2. 每个 section 都应该能作为一张独立长图导出。
3. 不要把多个无关主要模块混在同一个 section 里。
4. 但如果是同一个实验，实验文字和对应图表必须放在同一个 section 里。
5. 不要给 section 设置 overflow:hidden 导致截图裁切异常，除非确有必要。
6. section 外侧要保留页面背景，不要依赖透明背景。
7. 每个 section 的宽度应适配 390px 主容器。
8. 不要使用 position: fixed。
9. 不要使用需要用户交互才显示的内容。
10. 不要使用懒加载图片。
11. 所有图片使用普通 <img> 标签。
12. 图片必须设置 display:block; width:100%; height:auto;。
13. 所有图片直接写 `<img src="assets/N.png">`，不生成占位区域，不写占位文字。
14. 不要把图片设成背景图，因为后续替换和导出不方便。

## 十、内容改写要求

1. 请把论文内容改写成适合公众号读者阅读的科普表达。
2. 不要过度简化研究结论，避免夸大因果。
3. 保留关键概念、实验逻辑和核心发现。
4. 如果涉及统计结果，请简短解释其含义。
5. 中文表达要流畅、有节奏，但不要营销化。
6. 标题要有吸引力，但不要标题党。
7. 如果原文有多个研究或多个实验，请筛选最适合读者理解的几个重点呈现。
8. 每个实验要讲清楚"为什么做、怎么做、发现什么"，不要只写结论。
9. 如果实验之间是递进关系，要在每个实验开头说明它承接了上一个实验的什么问题。
10. 不要把实验编号写错，不要出现"实验3配图却是实验4图"的情况。
11. 语言要简洁朴实，不要啰嗦。一段话能说清的事不要拆成三段，不要在同一个意思上来回绕。
12. 避免使用中文破折号（——）。用句号、逗号或直接分句替代。
13. 用科普向的平实语言写作，避免过度渲染、煽情或鸡汤化表达。

## 十一、推荐文章结构

请优先采用下面的结构：

01 强视觉封面区
02 问题引入区
03 研究设计总览区
04 实验1：文字说明 + 对应图表
````
